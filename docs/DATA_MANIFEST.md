# Data manifest (manual downloads and file placement)

This review package is designed to let reviewers reproduce the manuscript outputs in a clear, traceable way.

Two reproducibility modes are supported:

1. Re-generate Figures 2–6 from the included result tables (no raw data downloads).
2. Re-run upstream analyses from public raw data (some downloads are manual; large datasets are not redistributed here).

### A1. Reference tables

- `data/references/ferroptosis_genes_expanded.csv`
- `data/references/tcm_targets_CHB.csv`

### A2. Minimal processed input

## B. Not redistributed (manual downloads required for full re-run)

### B1. GEO raw series matrix files (GSE14520)

If you want to re-run the GSE14520 parsing step, download the series matrix files from GEO and place them here:

- `data/raw/GSE14520/GSE14520-GPL3921_series_matrix.txt.gz`
- `data/raw/GSE14520/GSE14520-GPL571_series_matrix.txt.gz` (optional)

Source: GEO (GSE14520) series page, “Series Matrix File(s)”.

Script: `scripts/01b_download_GSE14520.R`

### B2. Platform annotation (GPL571)

For probe-to-gene mapping in immune/drug scripts:

- `data/raw/GSE14520/GPL571.annot.gz`

Source: GEO platform page (GPL571) annotation file.

Script: `scripts/03a_immune_infiltration_v2.R`

### B3. TCGA-LIHC (external validation)

This package does not redistribute TCGA data. You can either:

- generate `data/processed/TCGA_LIHC_expr.rds` and `data/processed/TCGA_LIHC_clinical.rds` using `scripts/02e_external_validation.R` (requires internet access), or
- prepare equivalent files yourself and place them under `data/processed/`.

### B4. GDSC2 drug sensitivity training data

Drug sensitivity analysis requires GDSC2 training data, which is not redistributed here:

- `data/processed/GSE14520_expr_symbol.rds` (generated by `scripts/03a_immune_infiltration_v2.R`)
- `data/references/GDSC/GDSC2_Expr.rds`
- `data/references/GDSC/GDSC2_Res.rds`

Script: `scripts/05c_drug_sensitivity_v2.R`

### B5. Molecular docking raw runs

CB-Dock2 is a web service; raw server output is not guaranteed to be reproducible byte-for-byte. This package provides the curated docking summary table used in the manuscript.

## C. Where to start

For reviewers, the simplest workflow is:

- `Rscript run_complete_pipeline.R`

This regenerates Figures 2–6 from the included result tables and writes to `plots/publication/`.
